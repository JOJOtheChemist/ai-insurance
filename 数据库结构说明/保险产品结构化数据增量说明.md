# 保险产品结构化数据增量说明

本文档基于 `all_products_viewer.html` 中的前端渲染逻辑，提取了不同险种在 `extend_info`（扩展信息）字段中的增量数据结构。这些数据用于在前端展示产品特有的权益和图表。

## 一、通用数据结构 (`extend_info` JSON)

所有产品共享以下基础 JSON 结构，但在具体内容上有所不同：

```json
{
  "extend_info": {
    "theme_color": "#D31145",  // 主题色 (Hex)
    "highlights": [            // 顶部关键指标卡片 (通常 4 个)
      { "label": "指标名称", "value": "指标值" }
    ],
    "coverage_list": [         // 核心保障权益列表
      { 
        "icon": "🏥",          // 图标 (Emoji)
        "title": "权益名称",    // 标题
        "desc": "权益描述",     // 简短描述
        "value": "保额/状态"    // 例如：全额、✅、100万
      }
    ],
    "table_data": {            // 费率表或利益演示表 (可选)
      "title": "表格标题",
      "headers": ["列1", "列2"],
      "rows": [
        ["值1-1", "值1-2"],
        ["值2-1", "值2-2"]
      ]
    }
  }
}
```

## 三、前端解析逻辑

在 `all_products_viewer.html` 中，前端通过以下逻辑解析上述数据：

1.  **Highlights**: 直接遍历 `extend_info.highlights` 数组生成卡片。
2.  **Coverage List**: 遍历 `extend_info.coverage_list`。如果没有该字段，前端不会显示“核心保障权益”板块。
    *   *注：前端使用了 `hexToRgba` 函数，根据 `theme_color` 动态生成图标的背景色。*
3.  **Table Data**: 检查 `extend_info.table_data`。如果存在，则根据 `headers` 和 `rows` 动态生成 HTML 表格。

这种设计使得后端可以通过简单的 JSON 配置，控制前端页面的核心展示内容，而无需修改前端代码。
