# 反馈 1：Filter 失效与数据结构化决策

**来源场景**: 场景一 (30岁程序员)

**问题描述**: 
测试日志显示 `filter(age_min=30)` 返回为空。
原因：数据库中 `age_range` 字段目前是非结构化的文本（如 "18-60周岁"、"三岁至六十四岁"），而 Filter 工具依赖数值比较，导致 SQL 筛选失效。

**用户疑问**:
> "这个地方是有问题的，这个年龄还有其他的一些，那要么就是我直接把文字全都返回，让AI去判断语义，还是说这里应该再结构化一下"

**深度分析**:
1.  **AI 语义判断 (As-Is)**:
    *   *机制*: 返回所有候选文本，Agent 逐个阅读。
    *   *瓶颈*: 无法处理大规模数据 (Recall Phase 失效)，Token 消耗大，性能差。
2.  **数据结构化 (Recommended)**:
    *   *机制*: 在 DB 新增 `min_age` (int) 和 `max_age` (int) 列。
    *   *行动*: 编写一次性清洗脚本，将 "18-60周岁" 解析为 `18` 和 `60`。
    *   *收益*: 支持毫秒级 SQL 筛选，大幅提升检索效率。

**待定决策**:
*   [ ] 推荐执行“数据结构化清洗”。等待用户确认。
